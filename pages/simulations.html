Empiryczny rozkład zmienności (absolutnych stóp zwrotu) indeksu NASDAQ100
Na początku chciałem po prostu sprawdzić, jak w rzeczywistości wygląda zmienność na indeksie NASDAQ100. Zamiast analizować kierunkowe zyski i straty, wyciągnąłem z danych wartości bezwzględne dziennych stóp zwrotu (absolute returns) z ponad 30 lat. Kod napisany w Pythonie pobiera dane, usuwa puste wartości (NaN) i generuje znormalizowany histogram w Plotly. Wykres idealnie pokazuje zjawisko grubych ogonów (fat tails). Ekstremalne ruchy rynkowe rzędu kilku procent zdarzają się dużo częściej, niż wynikałoby to z teoretycznego rozkładu normalnego. Kod pisany bez pomocy AI.

DATA PIPELINE & OOP
Obiektowy pipeline do pobierania i przetwarzania notowań
Praca z surowymi danymi finansowymi bywa uciążliwa, a ręczne powtarzanie tych samych operacji dla dziesiątek tickerów po prostu mija się z celem. Żeby zachować porządek, zastosowałem programowanie obiektowe (OOP) i napisałem klasę Stock, której zadaniem jest hermetyzacja całego pipeline'u. Klasa łączy się z API yfinance, radzi sobie z kłopotliwymi, wielopoziomowymi indeksami (multi-index), wylicza klasyczne wskaźniki (SMA50, SMA200) jako nowe cechy i czyści szereg z brakujących wartości. Rozbicie tego kodu na dwie oddzielne komórki w Jupyter Notebooku to celowy zabieg – pierwsza przechowuje logikę, druga wywołuje proces. Dzięki temu generowanie i wizualizacja danych są w 100% powtarzalne, a zbadanie innej spółki sprowadza się do zmiany tickera oraz szeregu czasowego w jednej linijce kodu. To miniatura architektury, z której korzystam w moim bocie tradingowym do pompowania danych bezpośrednio do środowiska RL.

import yfinance as yf 
import pandas as pd 
import plotly.graph_objects as go
import plotly as plt 
import datetime as dt 

class Stock:
    def __init__(self, ticker, start, end):
        self.ticker = ticker
        self.start = start
        self.end = end 
        self.df = None
        
    def get_data(self):
        df: pd.DataFrame = yf.download(self.ticker, self.start, self.end, interval = '1d')
        self.df = df
        if self.df is not None and not self.df.empty:
            return self.df
        else:
            raise ValueError("Brak Danych lub niepoprawne ramy czasowe")
            
    def prepare_data(self):
        self.df = self.df.droplevel(1, 1)
        self.df['Date'] = self.df.index
        self.df['SMA50'] = self.df['Close'].rolling(50).mean()
        self.df['SMA200'] = self.df['Close'].rolling(200).mean()
        self.df = self.df.dropna()
        return self.df
        
    def chart_data(self):
        fig = go.Figure()
        fig.add_trace(go.Candlestick(
            x = self.df['Date'],
            open = self.df['Open'],
            high = self.df['High'],
            low = self.df['Low'],
            close = self.df['Close'],
            name = 'Notowania'
        ))
        fig.add_trace(go.Scatter(
            x = self.df['Date'],
            y = self.df['SMA50'],
            mode = 'lines',
            name = 'Średnia krocząca z 50 dni'
        ))
        fig.add_trace(go.Scatter(
            x = self.df['Date'],
            y = self.df['SMA200'],
            mode = 'lines',
            name = 'Średnia krocząca z 200 dni'
        ))
        fig.update_layout(
            title = f'{self.ticker} Notowania',
            xaxis_title = 'Data',
            template = 'plotly_dark',
            yaxis_title = 'Cena'
        )
        fig.show()

Uproszczona symulacja kreacji pieniądza przez banki komercyjne

SYMULACJA MAKROEKONOMICZNA
Algorytmizacja mnożnika kreacji pieniądza
Napisałem ten skrypt, aby na twardych liczbach i krok po kroku zrozumieć mechanizm kreacji pieniądza przez banki komercyjne. Aby uzupełnić wiedzę teoretyczną, postanowiłem przetestować ją w praktyce. Symulacja opiera się na dwóch kluczowych założeniach: stopie rezerwy obowiązkowej na poziomie 3% oraz mechanizmie, w którym każdy wykreowany kredyt po wydaniu natychmiast wraca do systemu bankowego jako nowy depozyt. Pętla while iteracyjnie potrąca rezerwę i pompuje nowy kapitał z powrotem do puli, aż wyczerpie się potencjał kredytowy. Ten prosty algorytm świetnie udowadnia matematycznie, jak działa mnożnik - z zaledwie 3000 zł pierwotnych depozytów (bazy monetarnej), system sztucznie kreuje 97 000 zł wirtualnego kapitału. Czysta mechanika rynkowa.
money_creation_sim.py

depozyt1: float = 1000.0
depozyt2: float = 1000.0
depozyt3: float = 1000.0
depozyt_kredytobiorcy: float = 0.0

stopa: float = 0.03
krok = 1
while ((depozyt1 + depozyt2 + depozyt3 + depozyt_kredytobiorcy) * (1 - stopa)) > depozyt_kredytobiorcy + 0.01:
    calosc = depozyt1 + depozyt2 + depozyt3 + depozyt_kredytobiorcy
    rezerwa = stopa * calosc
    nowy_kredyt = calosc - rezerwa - depozyt_kredytobiorcy
    depozyt_kredytobiorcy += nowy_kredyt
    
    print(f"Obrót {krok}:")
    print(f"  Suma w banku: {calosc:.2f} zł")
    print(f"  Zablokowana rezerwa: {rezerwa:.2f} zł")
    print(f"  Stan konta kredytobiorcy: {depozyt_kredytobiorcy:.2f} zł\n")
    
    krok += 1

print("-" * 50)
print("WYNIK KOŃCOWY:")
print(f"Suma depozytów pierwotnych (1+2+3): {depozyt1 + depozyt2 + depozyt3:.2f} zł")
print(f"Wykreowany depozyt kredytobiorcy: {depozyt_kredytobiorcy:.2f} zł")
print(f"Łączna kwota w systemie (calosc): {depozyt1 + depozyt2 + depozyt3 + depozyt_kredytobiorcy:.2f} zł")
